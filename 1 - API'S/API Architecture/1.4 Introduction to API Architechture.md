## ğŸ” Stateful vs Stateless APIs

- **Stateful**:
- Stores session info.
- More vulnerable to session-based attacks.
- Example: `SOAP API`, e-commerce cart stored server-side.

- **Stateless**:
- Every request is independent.
- No session tracking.
- More scalable and secure.
- Example: `REST APIs`.
